Topic 1 — Power BI Basics & Architecture

Power BI is a business intelligence and analytics platform from Microsoft. It enables users to connect to different data sources, clean and transform data, build visual reports, and publish interactive dashboards for decision-making.
The main components are:
• Power BI Desktop – Report building & data modeling
• Power BI Service – Cloud sharing & collaboration
• Power BI Mobile – View dashboards on phone
• Power BI Gateway – Connect on-prem data to cloud
• Power BI Report Builder – Paginated reports (pixel-perfect)
Power BI Desktop is for developing reports, creating data models, and writing DAX.
Power BI Service is for publishing, sharing, refreshing data, scheduling updates, and collaboration.
A report is a multi-page collection of visualizations created in Power BI Desktop. It allows interactions such as filtering, drill-down, highlighting, and cross-report navigation.
A dashboard is a single-page canvas created in Power BI Service using pinned visuals. It gives high-level KPIs and can combine visuals from multiple reports.
A dataset is the connected or imported data used to build visuals. It includes tables, relationships, measures, and transformations—the entire semantic model.
Import Mode:
• Data is stored inside Power BI memory (VertiPaq)
• Very fast performance
• Best for analytics

DirectQuery Mode:
• Data remains in the source database
• Executes live queries
• Supports large datasets and real-time requirements
Gateway securely connects on-premises data sources (SQL Server, local Excel files, shared folders) with the Power BI Service. It enables scheduled refresh and real-time connectivity.
Workspaces are collaborative environments in Power BI Service where teams can store datasets, reports, dashboards, dataflows, and apps. They act like project folders.
Power BI offers 4 types of filters:
• Visual-level filters
• Page-level filters
• Report-level filters
• Drill-through filters
The semantic model contains all tables, relationships, hierarchies, and calculations used for reporting. It acts as the logical layer between raw data and visuals, ensuring consistency.
Common reasons:
• High-cardinality columns
• Complex DAX calculations
• Slow DirectQuery source performance
• Too many visuals per page
• Dataset not optimized
• Hardware differences between your PC and Power BI Service
Possible causes:
• Scheduled refresh failed
• Gateway offline or misconfigured
• Browser cache
• Source credentials expired
• Import mode report not manually refreshed
Methods:
• Share from workspace (requires Pro)
• Publish an App
• Assign Viewer, Member, or Contributor roles
• Apply Row-Level Security for confidential data
Real-time dashboards can be built using:
• DirectQuery connections
• Streaming datasets
• Push datasets via API
• Power Automate streaming triggers
A tile is a single visual element pinned to a dashboard—such as a chart, KPI, card, map, or image.
License options:
• Free
• Power BI Pro
• Power BI Premium Per User (PPU)
• Power BI Premium (capacity-based)
Q&A allows users to ask questions in natural language like “Show sales by country.” Power BI automatically generates visuals based on the dataset.
VertiPaq is Power BI’s in-memory columnar engine. It compresses data and performs very fast analytical queries, making Import mode extremely efficient.
The end-to-end workflow:
1. Connect to data
2. Transform & clean in Power Query
3. Build data model
4. Create DAX calculations
5. Build visuals & report
6. Publish to Service
7. Schedule refresh
8. Share dashboards & collaborate
