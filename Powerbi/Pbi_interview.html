Topic 1 — Power BI Basics & Architecture

Power BI is a business intelligence tool from Microsoft that helps users connect to data, clean it, build visuals, and create interactive dashboards. It enables decision-makers to understand trends, patterns, and performance through easy-to-interpret visuals.
Power BI consists of 5 main components: • Power BI Desktop – for data modeling and report building • Power BI Service – for publishing and sharing • Power BI Mobile – for viewing on phones • Power BI Gateway – connects on-premise data to the cloud • Power BI Report Builder – for paginated reports
Power BI Desktop is used to build data models and reports. Power BI Service is the cloud platform used to publish, share, refresh, and collaborate.
A report is a multi-page collection of visualizations created in Power BI Desktop. Reports allow filtering, drilling down, and interactive exploration of data.
A dashboard is a single-page interface created only in the Power BI Service. It displays key visuals pinned from various reports for quick monitoring.
A dataset is the collection of tables, relationships, and data that is used to build reports. It is the underlying semantic model that powers visuals.
Import Mode loads data into Power BI memory (VertiPaq), providing extremely fast performance. DirectQuery keeps data in the source database and queries it live, suitable for real-time needs.
A gateway connects on-premise data (SQL Server, Excel files in local systems) to Power BI Service for scheduled refresh and real-time access. It ensures secure data transfer between local servers and the cloud.
Workspaces are cloud collaboration spaces where teams store datasets, reports, dashboards, and apps. They act as shared folders for BI development and publishing.
Power BI provides 4 types of filters: • Visual Level Filters • Page Level Filters • Report Level Filters • Drillthrough Filters
A semantic model is the structured data model containing tables, relationships, measures, and calculations. It provides consistent logic for all visuals and reports using that dataset.

Common causes are failed scheduled refresh, gateway issues, cached visuals, expired credentials, or using Import mode without refreshing manually.
Reports can be shared through workspace access, sharing links (Pro license), or by publishing an App. Row-Level Security (RLS) ensures confidential data protection.
Real-time dashboards are achieved using DirectQuery, streaming datasets, push datasets, or REST API. These methods allow immediate updates without manual refresh.
A tile is a single visual object pinned to a dashboard such as a chart, map, KPI, or card.
Power BI offers Free, Pro, Premium Per User (PPU), and Premium (capacity-based) licenses. Premium provides dedicated capacity and advanced enterprise features.
Q&A is a natural-language feature allowing users to type questions like “Sales by region” and instantly generate visuals based on the dataset.
Power BI uses the VertiPaq in-memory engine, which compresses data and processes it column-wise, enabling extremely fast performance for analytical queries.
The typical workflow is: 1) Get Data → 2) Transform Data → 3) Build Model → 4) Create Visuals → 5) Publish → 6) Share & Refresh. This completes the end-to-end BI development cycle.
