ATLAS 1 — Machine Learning Fundamentals

Machine Learning is the ability of a system to learn patterns from historical data and make predictions or decisions on new, unseen data without being explicitly programmed.

Interview expectation: You must say prediction + learning from data, not definitions.
In traditional programming, rules are written by humans and data is processed using those rules. In Machine Learning, data and outcomes are provided to the algorithm, and the system learns the rules itself.

Real example: Spam filtering cannot be rule-based effectively — ML adapts automatically.
Statistics focuses on inference, hypothesis testing, and explaining relationships. Machine Learning focuses on prediction accuracy and automation at scale.

Interview trap: Saying “ML is advanced statistics” is ❌.
Machine Learning should not be used when:
  • Simple rule-based logic is sufficient
  • Very little data is available
  • No clear pattern exists
  • Business rules frequently change
This is a GOLD interview answer.
  • Supervised Learning — labeled data (regression, classification)
  • Unsupervised Learning — unlabeled data (clustering)
  • Semi-supervised Learning — partial labels
  • Reinforcement Learning — learning via rewards
Interviewers mainly expect clarity on the first two.
A problem is suitable when:
  • Historical data exists
  • Patterns can be learned
  • Future predictions are valuable
  • Errors have acceptable tolerance
ML is not magic — it needs learnable structure.
Data leakage occurs when information from the future or target leaks into training data. It causes unrealistically high accuracy and complete failure in production.

Example: Using post-loan approval data to predict loan approval.
This is called distribution shift. The model learns patterns that do not exist in real-world data, leading to poor generalization and production failure.
Yes. Accuracy alone does not reflect business value. In imbalanced datasets, predicting the majority class can give high accuracy but zero usefulness.

Example: Fraud detection.
Common reasons:
  • Data drift
  • Poor data quality
  • Overfitting
  • Wrong metric optimization
  • Changing business behavior
Interviewers love this question.
Generalization is the ability of a model to perform well on unseen data, not just on training data. Good models generalize — bad models memorize.
By:
  • Using proper train/validation/test split
  • Cross-validation
  • Monitoring performance after deployment
There is no guarantee — only controlled risk.
